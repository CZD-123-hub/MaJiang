# 基于知识项目组前两个任务实现说明

## 任务一：手牌组合计算

实现位置：

- `mahjong_ai/tiles.py`
- `mahjong_ai/hand_split.py`

本实现沿用 `v5_rh` 里的 16 进制麻将编码：

- 万子：`0x01-0x09`
- 条子：`0x11-0x19`
- 筒子：`0x21-0x29`
- 字牌：`0x31-0x37`

`split_hand(hand)` 会递归枚举平胡拆分方式。每个组合包含：

- `triplets`：刻子集合
- `sequences`：顺子集合
- `pairs`：AA 搭子 / 将牌候选
- `taatsu`：AB、AC 搭子
- `leftovers`：剩余单张
- `shanten`：任务书定义的向听数，胡牌为 0，听牌为 1

向听数计算采用平胡结构：4 组面子 + 1 对将。算法先统计已完成面子，再统计可用搭子，最后按缺少的面子数和将牌数计算距离胡牌还差几张牌。

## 任务二：博弈树扩展和评估

实现位置：

- `mahjong_ai/search_tree.py`

当前实现的是一层“出牌-摸牌”搜索树：

1. 根节点是当前 14 张手牌。
2. 第一层枚举每一种候选弃牌。
3. 第二层枚举弃牌后可能摸到的牌。
4. 每个摸牌子节点重新计算向听数。
5. 对每个弃牌节点进行评分，选择评分最高的出牌。

评分主要考虑：

- 弃牌后的向听数，向听越少越好。
- 摸牌后能降低向听的有效进张数量。
- 有效牌剩余张数，剩余越多越好。
- 当前拆分中已有面子、搭子的数量。
- 孤张数量，孤张越多评分越低。

## 全扩展与剪枝

如果完全扩展，节点会快速膨胀：每个弃牌后都可能有 34 类摸牌，每个摸牌后又能继续枚举弃牌、吃碰杠、对手出牌和未知牌墙状态。完整扩展更全面，但计算量很大。

如果过度剪枝，只保留少量组合或少量有效进张，速度会变快，但可能漏掉短期向听数不优、长期收益更高的路线。

本实现采用折中方式：

- 第一层保留全部候选弃牌。
- 第二层枚举全部可摸牌。
- 组合拆分默认按向听数、面子数、搭子数排序。
- 需要提速时，可以通过 `split_hand(limit=...)` 或 `expand_discard_tree(max_draws=...)` 只保留较优分支。

这对应任务书里“选取哪些组合进行博弈树扩展、全扩展和剪枝如何平衡”的要求。
