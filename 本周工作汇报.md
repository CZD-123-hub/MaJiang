## 一、本周工作目标

本周主要围绕 v5 上饶麻将 AI 源代码进行阅读和分析，重点理解其平胡牌型组合拆分逻辑、平胡博弈树扩展评估逻辑，以及完整的出牌决策和动作调用决策流程。在此基础上，结合后续九江红中麻将 AI 开发需求，设计一套适合九江红中的平胡组合拆分思路和平胡博弈树评估方案，并分析已有代码在牌型组合拆分方面存在的问题和优化方向。

## 二、v5 代码阅读内容

本周重点阅读了 v5 上饶麻将 AI 中以下核心模块：

1. `action_recommendation.py`
   主要负责外部推荐接口，包括 `recommend_card(...)` 出牌推荐和 `recommend_op(...)` 动作推荐。

2. `get_score_dict.py`
   负责四种牌型的评分调度，包括平胡、十三烂、九幺、七对。

3. `Node_PH.py`
   负责平胡相关逻辑，包括 `PingHu` 平胡组合拆分、`SearchTree_PH` 平胡搜索树扩展和 `Fan_PH` 番数计算。

4. `lib_MJ.py` 和 `Utils.py`
   提供基础麻将工具函数，包括牌编码转换、向听数计算、搭子转面子、孤张转面子、剩余牌统计等。

通过阅读可以看出，v5 的整体决策思想是：

```text
先进行牌型组合拆分
-> 计算各牌型向听数
-> 选择接近最小向听数的牌型
-> 对候选牌型进行搜索树扩展
-> 根据摸牌概率、进张权重和番数计算路径收益
-> 汇总每张候选弃牌评分
-> 返回评分最高的出牌或动作
```

## 三、v5 平胡组合拆分逻辑理解

v5 中平胡组合拆分的核心函数是：

```python
PingHu.pinghu_CS2()
```

它的主要流程为：

```text
输入当前手牌
-> 去除宝牌
-> 按万、条、筒、字牌拆分
-> 万、条、筒分别递归拆分刻子、顺子、对子、搭子
-> 字牌单独拆分刻子、对子、孤张
-> 合并各花色拆分结果
-> 过滤不合理组合
-> 计算每种组合的向听数
-> 返回候选平胡组合
```

组合结果使用类似以下结构表示：

```text
[kz, sz, aa, ab, xts, leftCards]
```

其中：

- `kz` 表示刻子集合；
- `sz` 表示顺子集合；
- `aa` 表示对子集合；
- `ab` 表示搭子集合；
- `xts` 表示向听数；
- `leftCards` 表示剩余孤张。

v5 的拆分方式比较完整，能够枚举多种可能组合，并且会过滤一些明显不合理的拆法。例如 `3、4、5` 不应被拆成 `3、5` 搭子加 `4` 孤张，而应优先保留为顺子。

## 四、自己设计的平胡组合拆分方案

针对后续九江红中麻将 AI，我设计的平胡组合拆分思路是：

```text
先拆普通牌，再处理红中万能牌
```

具体流程如下：

```text
输入手牌
-> 单独统计红中数量
-> 从手牌中移除红中
-> 普通牌按万、条、筒、字牌拆分
-> 数牌递归拆分刻子、顺子、对子、搭子、孤张
-> 字牌只拆刻子、对子、孤张
-> 合并普通牌组合
-> 使用红中补将、补顺子、补刻子
-> 计算向听数和组合评分
-> 保留较优组合
```

该方案中，红中不直接参与普通牌自然拆分，而是作为万能资源在后续补缺：

1. 可以用红中补对子作为将牌。
2. 可以用红中补搭子成为顺子。
3. 可以用红中补对子或单张成为刻子。
4. 如果有四张红中，则优先判断四红中直接胡牌。

组合排序时，不只看向听数，还需要综合考虑：

- 完整面子数量；
- 对子数量；
- 两面、嵌张、边张等搭子质量；
- 孤张数量；
- 红中消耗数量；
- 有效进张剩余张数。

这样设计的好处是普通牌结构和红中万能牌逻辑分开，后续适配九江红中规则会更清晰。

## 六、v5 平胡博弈树扩展评估逻辑理解

v5 的平胡搜索树由 `SearchTree_PH` 和 `Node_PH` 实现。

每个节点保存当前平胡路径中的状态，包括：

- 已完成刻子 `AAA`；
- 已完成顺子 `ABC`；
- 将牌 `jiang`；
- 搭子集合 `T2`；
- 孤张集合 `T1`；
- 需要摸到的牌 `taking_set`；
- 进张权重 `taking_set_w`；
- 剩余宝牌数量 `king_num`。

搜索树扩展流程可以概括为：

```text
根据平胡拆分组合生成根节点
-> 先扩展将牌
-> 判断是否已经满足胡牌结构
-> 如果未胡，继续扩展搭子和孤张
-> 使用转换表将 T2、T1 扩展成 T3
-> 搜索到胡牌叶子节点
-> 计算胡牌路径收益
-> 将路径收益回传给候选弃牌
```

v5 的评分核心是：

```text
路径分数 = 番数 × 摸牌概率 × 进张权重
```

其中番数由 `Fan_PH.fanDetect()` 计算，摸牌概率来自剩余牌统计，进张权重来自搭子或孤张转面子的转换表。

## 七、自己设计的平胡博弈树评估方案

针对九江红中麻将，我设计的平胡博弈树评估方案更偏向从候选弃牌出发：

```text
当前手牌
-> 枚举每一张候选弃牌
-> 模拟打出后得到 13 张手牌
-> 枚举下一次可能摸到的牌
-> 重新进行组合拆分和向听数计算
-> 根据局面质量给每张候选弃牌评分
-> 选择评分最高的弃牌
```

评分可以由以下部分组成：

```text
弃牌得分 =
向听数基础分
+ 有效进张分
+ 剩余牌数量分
+ 组合结构分
+ 红中利用分
+ 规则收益分
- 孤张惩罚
- 红中过度消耗惩罚
- 危险牌惩罚
```

该方案初期可以先做两层搜索，即“弃牌 -> 摸牌”，保证能稳定给出决策。后续如果计算量允许，再扩展到多层搜索。

相比 v5 从胡牌叶子节点回传弃牌价值的方式，我的方案实现更直观，调试更方便，也更适合九江红中 AI 初版开发。

## 八、上饶麻将出牌决策和调用决策逻辑

v5 的上饶麻将决策分为出牌决策和动作调用决策两部分。

### 1. 出牌决策

出牌决策入口是：

```python
recommend_card(...)
```

完整流程为：

```text
recommend_card(...)
-> 更新轮数、宝牌等全局信息
-> 根据弃牌、副露、手牌统计剩余牌 LEFT_NUM
-> 根据剩余牌数量计算自摸概率 T_SELFMO
-> 调用 get_score_dict(...)
-> 计算平胡、十三烂、九幺、七对四种牌型向听数
-> 选择接近最小向听数的牌型进入搜索
-> 分别计算每种牌型下的候选弃牌评分
-> 合并不同牌型下同一张弃牌的评分
-> 返回总评分最高的牌
```

因此，v5 的出牌决策不是单一牌型判断，而是多牌型综合评分。

### 2. 动作调用决策

动作调用决策入口是：

```python
recommend_op(...)
```

完整流程为：

```text
recommend_op(...)
-> 如果可以胡，直接胡
-> 计算不操作时的最高评分 score_pre
-> 如果是自己回合，判断暗杠和补杠
-> 如果是别人回合，判断明杠、吃、碰
-> 枚举所有可操作组合
-> 模拟操作后的手牌和副露
-> 重新调用 get_score_dict(...) 计算操作后最高评分
-> 如果操作后最高评分 > 操作前评分 × 1.05，则执行该操作
-> 否则选择过
```

其中 `1.05` 是操作收益阈值，表示操作后评分至少要比操作前高 5%，才认为该操作值得执行。

整体来看，v5 的调用决策不是简单地“能碰就碰、能吃就吃”，而是通过操作前后评分比较来判断操作是否真的提升手牌价值。

## 九、已有代码问题与优化方向

从“各牌型组合拆分逻辑是否合理，进行优化”这个方向看，已有代码主要有以下问题：

1. **拆分组合数量较多**
   递归枚举会产生大量候选组合，可能增加搜索树计算压力。

   优化方案：按向听数、面子数量、搭子质量、孤张数量进行排序，只保留前若干个较优组合。

2. **搭子价值区分不够细**
   两面搭子、嵌张搭子、边张搭子的实际价值不同，但现有逻辑区分不够明显。

   优化方案：为不同搭子设置不同权重，两面搭子优先级高于嵌张和边张。

3. **孤张价值没有充分区分**
   中张孤张比边张和字牌更容易形成有效搭子。

   优化方案：根据牌值区分孤张价值，中张数牌优于边张数牌，字牌孤张价值较低。

4. **特殊牌规则耦合较强**
   v5 主要围绕上饶麻将宝牌规则设计，不能直接适配九江红中。

   优化方案：将特殊牌处理抽象成独立模块，上饶使用宝牌策略，九江红中使用红中万能牌策略。

5. **九江红中特殊规则需要重新适配**
   九江红中不可吃，红中不能碰杠，四红中可直接胡。

   优化方案：删除吃牌分支；动作判断时禁止红中碰杠；胡牌判断入口加入四红中特判。
