## 优化建议

### 高优先级优化

#### 1. 调查座位差异原因 🔍
**现象**: 同一算法在不同座位表现差异达0.78分/局

**可能原因**:
- 代码中是否有依赖座位ID的逻辑？
- 是否有使用固定随机种子导致特定座位有优势？
- 座位2是否在特定庄位条件下有优势？

**建议**:
- 检查代码中所有使用`player_id`或`seat_id`的地方
- 增加更多对局（100+局）验证是否是统计波动
- 交换座位重新测试，看座位2优势是否转移

#### 2. 提升进攻侵略性 ⚔
**现象**: 防守很好，但胜率一般

**建议**:
- 当前策略可能过于保守，追求安全而非速度
- 在向听数较低时（1-2向听），可以适当降低防守权重
- 优化博弈树评估：增加"有效进张数"和"进张剩余数量"的权重
- 考虑"听牌类型价值"：两面听 > 对倒 > 边张 > 嵌张

#### 3. 多轮博弈树搜索深度 🌲
**当前问题**: 可能只搜索1-2层（弃牌-摸牌）

**建议**:
- 在关键决策点（1向听→听牌），增加搜索深度
- 考虑对手可能的反应（对手也在追求听牌）
- 实现动态搜索深度：远听时浅搜，近听时深搜

### 中优先级优化

#### 4. 改进危险牌判断 🎯
虽然点炮率已经很低，但仍可优化：
- 根据对手碰牌/杠牌推断听牌类型
- 统计对手出牌模式（早巡打字牌→可能做清一色）
- 实现"筋牌理论"：对手打5，则1-9相对安全

#### 5. 加入番种评估 💰
**当前**: 只考虑胡牌，不考虑番数

**建议**:
- 识别高价值牌型：清一色、对对胡、混一色
- 在牌型接近时，优先选择高番型
- 权衡速度与番数：1向听平胡 vs 2向听清一色

#### 6. 动态策略调整 📊
根据当前分数动态调整策略：
- 大幅领先时：采用保守策略，避免点炮
- 大幅落后时：采用激进策略，追求高番
- 接近平局时：平衡进攻与防守

### 低优先级优化

#### 7. 对手建模
- 统计每个对手的打法习惯
- 识别"炮台型"对手（高点炮率）
- 针对不同对手调整策略

#### 8. 记忆与学习
- 记录历史对局中的成功/失败决策
- 建立"案例库"用于相似局面参考

```python
def evaluate_wait_type(self, waits):
    """评估听牌类型价值"""
    # 两面听（如等4-7的5-6）
    if is_double_sided(waits):
        return 10
    # 对倒（两对等单张成将）
    elif is_pair_wait(waits) and len(waits) >= 2:
        return 8
    # 嵌张（等5的3-4-6）
    elif is_middle_wait(waits):
        return 3
    # 边张（等3的1-2）
    elif is_edge_wait(waits):
        return 2
    return 5
```

### 危险牌评估

```python
def evaluate_danger(self, tile, opponent_state):
    """评估打出某张牌的危险度"""
    danger = 0
    
    # 对手碰过的牌，相邻牌危险
    for peng in opponent_state['pengs']:
        if is_adjacent(tile, peng):
            danger += 30
    
    # 对手从未打过的牌，危险
    if tile not in opponent_state['discards']:
        danger += 20
    
    # 中张牌（4-6）危险度高
    if is_middle_tile(tile):
        danger += 10
    
    return danger
```
